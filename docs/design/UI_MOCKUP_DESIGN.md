## 6.5 원문 보기 UX (URL/PDF)

### 6.5.1 source_type별 동작 정의

| source_type | 버튼 | 클릭 동작 | 이유 |
|-------------|------|----------|------|
| `url` | 🔗 원문 보기 | **새 탭**에서 원본 URL 열기 | CORS/X-Frame-Options 회피 |
| `pdf` | 📄 PDF 원문 보기 | **새 탭**에서 R2 URL 열기 | 브라우저 내장 PDF 뷰어 활용 |
| `text` (또는 없음) | 표시 안함 | - | 원문 없음 |

### 6.5.2 URL 원문 보기

```
사용자 클릭: [🔗 원문 보기]
    │
    ▼
window.open(doc.source_url, '_blank')
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│  [새 브라우저 탭]                                                │
│                                                                 │
│  https://example.com/react-tutorial                             │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                     원본 웹페이지                          │ │
│  │                                                           │ │
│  │   (React 기초 튜토리얼 원본 콘텐츠)                        │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

**장점**:
- X-Frame-Options 차단 사이트도 정상 표시
- 원본 사이트의 모든 기능 (로그인, 댓글 등) 사용 가능
- 추가 구현 불필요

### 6.5.3 PDF 원문 보기

```

**장점**:
- Chrome/Edge 내장 PDF 뷰어 활용 (추가 라이브러리 불필요)
- 확대/축소, 페이지 이동, 다운로드, 인쇄 기본 제공
- R2 공개 URL로 직접 접근 가능

### 6.5.4 버튼 스타일

```jsx

    rel="noopener noreferrer"
    className="inline-flex items-center gap-2 px-4 py-2
               bg-blue-50 text-blue-600 rounded-lg
               hover:bg-blue-100 transition text-sm"

    🔗 원문 보기
  </a>
)}

{/* PDF 원문 보기 */}
{doc.source_type === 'pdf' && doc.source_file && (
  <a
    href={doc.source_file}
    target="_blank"
    rel="noopener noreferrer"
    className="inline-flex items-center gap-2 px-4 py-2

               hover:bg-gray-200 transition text-sm"

    📄 PDF 원문 보기
  </a>
)}
```

## 8. PRD 적합성 검토

### 8.1 PRD 요구사항 vs 현재 구현

| PRD 요구사항 | 구현 상태 | 적합성 | 비고 |
|-------------|-----------|--------|------|
| **US-01**: 메모/녹음 업로드 → 1분 내 매뉴얼 생성 | ⚠️ 부분 구현 | 70% | 텍스트/URL 지원, 녹음(STT) 미구현 |
| **US-02**: 위키 형태 에디터, AI 수정 제안 | ⚠️ 부분 구현 | 50% | 기본 편집만 지원, AI 수정 제안 없음 |
| **US-03**: 학습 완료 후 5~10문항 퀴즈 | ✅ 구현됨 | 100% | 4문항 퀴즈, 10개 문항 중 랜덤 선택 |
| **US-04**: 오답 노트, 관련 본문 하이라이팅 | ❌ 미구현 | 0% | 오답 시 정답만 표시, 본문 링크 없음 |
| **US-05**: 팀별/개인별 진척도 그래프 | ✅ 구현됨 | 100% | AdminDashboard 통계 탭 |

### 8.2 핵심 기능 PRD 적합성

| 기능 | PRD 명세 | 현재 구현 | 적합성 |
|------|----------|-----------|--------|
| **다중 포맷 입력** | 텍스트, PDF, Word, PPT, 음성, 이미지 | 텍스트, URL, PDF(예정) | 40% |
| **자동 구조화** | 서론-본론-결론, Step-by-Step | 섹션 분리, Step 분할 | 80% |
| **역질문 생성** | 부족한 맥락 발견 시 멘토에게 역질문 | ❌ 미구현 | 0% |
| **퀴즈 자동 생성** | OX, 4지선다, 단답형, 시나리오형 | 4지선다만 | 25% |
| **난이도 조절** | 상/중/하 난이도 | ❌ 미구현 | 0% |
| **랜덤 출제** | AI가 문맥을 비틀어 새로운 문제 생성 | 랜덤 셔플만 | 30% |
| **멘티 현황** | 진도율, 퀴즈 평균, 취약 파트 분석 | ✅ 구현됨 | 100% |
| **멘토 기여도** | 자료 업로드 수, 멘티 피드백(별점) | 업로드 수만 | 50% |

### 8.3 UI/UX 적합성

| 영역 | 평가 | 개선 필요 사항 |
|------|------|---------------|
| **일관성** | ⚠️ 보통 | Spinner, Button 스타일 통일 필요 |
| **접근성** | ⚠️ 보통 | aria 속성 일부만 적용 (Pagination) |
| **반응형** | ✅ 양호 | Tailwind grid 사용, 모바일 지원 |
| **피드백** | ✅ 양호 | Toast 알림 사용 |
| **로딩 상태** | ✅ 양호 | Spinner, 진행률 표시 |

### 8.4 개선 권장사항

#### 우선순위 1 (Critical)
1. **오답 노트 기능** - PRD US-04 미구현
2. **다양한 퀴즈 유형** - 현재 4지선다만 지원

#### 우선순위 2 (High)
3. **PDF 업로드** - 현재 "구현 예정" 상태
4. **난이도 조절** - 퀴즈 난이도 선택 없음
5. **멘토 피드백(별점)** - 멘토 기여도에 피드백 없음

#### 우선순위 3 (Medium)
6. **음성 입력(STT)** - PRD에는 있으나 미구현
7. **AI 수정 제안** - 편집 시 AI 제안 없음
8. **UI 패턴 통일** - Spinner, Button 스타일
